# xox-model

面向小型经营团队的测算、记账、预实分析与版本化管理平台。

## 仓库结构

- `apps/web`：React + Vite 前端
- `apps/api`：FastAPI + SQLAlchemy 后端
- `docs`：架构、项目规划、接口说明、验收清单、运维说明
- `infra/scripts`：部署与辅助脚本

## 功能范围

- 认证：注册、登录、退出登录、注销账号
- 测算：可编辑草稿、自动保存、发布版本、历史回滚、公开分享
- 测算口径：支持单独配置拍立得损耗率，默认 10%，按张成本会按售出张数乘以 `1 + 损耗率` 计入
- 成本编辑：默认专项成本包含 `舞台视觉 / 原创 / 服装 / 演出收费 / 化妆 / 推流 / 聚餐 / 团建 / 耗材`，旧工作区缺少的默认列会自动补齐
- 记账：基于当前草稿的预测科目录入实际收入与成本分录，无需先发布正式版
- 预实分析：按期间与科目对比当前草稿计划和实际结果
- 分享：为不可变发布版生成只读公开链接，并支持撤销

## 本地开发

### 前端

```bash
npm.cmd install
npm.cmd run dev:web
```

### 后端

```bash
python -m pip install -e ./apps/api
python -m uvicorn app.main:app --app-dir apps/api --reload
```

## Linux 部署

在 Linux 服务器上可以直接使用 `infra/scripts/deploy-linux.sh` 完成前后端部署。这个脚本不会修改 nginx 配置，而是：

- 安装 Node.js 20 和 Python 3.11+
- 构建前端并安装后端依赖
- 创建 `xox-model-web` 和 `xox-model-api` 两个 systemd 服务
- 让前端静态服务在同源下代理 `/api/*` 到本机 FastAPI

在仓库工作树里执行：

```bash
sudo bash infra/scripts/deploy-linux.sh
```

如果脚本是单独拷到目标机器当前目录的，需要额外提供仓库地址：

```bash
sudo REPO_URL=<git-url> bash ./deploy-linux.sh
```

常用参数：

```bash
sudo WEB_PORT=4173 API_PORT=8000 PUBLIC_ORIGIN=http://127.0.0.1:4173 bash infra/scripts/deploy-linux.sh
sudo RUN_TESTS=1 bash infra/scripts/deploy-linux.sh
```

## 说明

- 前端放在 `apps/web` 下，避免仓库随着功能增长而把所有文件堆在根目录。
- 后端围绕“可变草稿、不可变发布版、按期间记账、按当前草稿做预实分析”这条主线设计。
- 公开分享只允许针对发布版，确保外部链接不会被后续草稿修改污染。
- 审计日志覆盖认证、工作区、分享和账务关键动作。
- 发布时会同时固化月度事实表和行项目事实表，便于后续对账与分析。
- 账期、科目和预实分析会跟随当前草稿实时同步；发布和回滚只是在同一套账务口径上固化或切换版本。

## 体验说明

- 认证页聚焦登录与注册主任务，不再用大段说明抢占注意力。
- 登录后的账号、工作区、版本入口统一收纳在左侧产品壳，避免顶层横向账号条破坏层级。
- 移动端壳层改为紧凑工具条，四个主步骤收成两列宫格，主内容在首屏即可露出。
- 主导航改成动作导向的业务步骤，桌面端收成轻量侧边步骤列，不再让导航像第二个内容区。
- 收入记账改成“按团员的表格账本”，以团员为行，主录入列收敛成 `线下张数 / 线上张数 / 本次收入 / 自动提成`，桌面端尽量保持单行，不再先选科目。
- 收入口径统一收敛到团员台账，不再保留独立的“线上 / 渠道回款”第二录入区；线上和线下都在同一成员行里录，系统再把两路收入映射回预算科目。
- 记账页补了“其他收支”入口，挂账科目用下拉选择，可直接记 `线下营收 / 线上营收 / 退费退款`，用于商业演出、一次性回款、赞助回款和退费回款这类不按团员台账录入的记录。
- 收入账本把“业务发生日”和“记账时间”拆开记录：历史列表展示发生日，明细保留实际过账时间，便于处理延迟回款和补录。
- 团员收入过账时会自动按当前草稿里的提成规则生成“成员提成”成本分录，不允许再手工重复录同一笔提成。
- 桌面端记账表格改成高密度台账：团员信息压成单行、时间列改为发生日加记账时间同行展示，并移除“待补收入”这类非必要主列，把宽度优先让给 `线下张数 / 线上张数 / 本次收入 / 自动提成` 这些真正用于录账的列。
- `成员底薪 / 成员路费` 这类挂成员的支出改成成员台账表格式录入，不再要求先点成员卡片再逐笔填写。
- 记账页的金额 / 张数输入在零值时默认显示为空，不再预填 `0`，录入时可以直接开始输入。
- 团员收入台账和成员支出台账都补了“一键入账”，会按当前草稿顺序逐行过账，减少逐个点击的重复操作。
- 团员收入台账和成员支出台账头部的“业务发生日 / 计划 / 已记 / 待入账”摘要统一收成单行胶囊，避免标题区再堆成两层。
- 团员收入台账和其他收支顶部都统一用“本期已记”口径；其中团员收入台账只统计挂到成员的收入分录，“其他收支”只统计无关联对象的独立收支，不再把团员台账里挂到同科目的分录混进去。
- 进入“本期账本”时会优先按“业务发生日”的月份定位到账期；首次进入三处“业务发生日”默认今天，后续无论你切账期还是直接改日期，账期和业务发生日月份都会保持同步。后端也会把显式传入的 `业务发生日` 作为归账月份，避免出现 `4 月日期 + 3 月账期` 这种错位；如果接口没有显式传日期，则继续尊重当前账期。
- “本期账本”顶部不再显示 `计划收入 / 计划成本 / 实际收入 / 实际成本` 这组汇总条，首屏只保留账期切换和锁定操作。
- 成员底薪 / 路费台账不再展示“对方单位 / 备注”补充区，保持纯表格录入。
- 成本编辑里的“月份”首列会尽量收窄，每个月的恢复默认按钮固定在月份格右上角，不再把首列撑宽或落到中间位置。
- 成本编辑的“专项与耗材”列宽会尽量收紧，`单价 / 场次`、`单价 / 系数` 的展示统一改成 `单价 × 场次`、`单价 × 系数`。
- `专项与耗材`、`训练与补充`、`收入引擎`、`团队成员假设`、`运营员工配置`、`股东投资` 这些录入表统一收成更紧凑的行高，减少竖向空白；表内月份/默认标签也同步压小，避免在高密度表格里显得发胀。
- 上面这些录入表里的输入框字体统一压到约 `11px` 这一档；同时修复了全局 `font: inherit` 对按钮和输入框字号的覆盖，局部字号设置现在会真实生效。
- 高密度表格现在共用一套 `HeaderCell / BodyCell / DenseFieldInput / DenseFieldSelect`，统一维护表头字号、对齐、行高、边框，以及表内 `number / date / select` 的字号、字重和框高，不再在各页散落重复样式。
- `02 调模型 / 03 记实际 / 04 看偏差` 这三条主流程已经统一到同一套工作台字体语言：页标题 `16px`、区块标题 `14px`、摘要值和动作文字 `12px`，其余输入、字段标题、表头、正文和辅助说明默认收敛到 `11px`。
- 成本科目仍保留科目驱动，但会同步显示“计划 / 已记 / 余量”，用户不用再自己心算还差多少。
- 支出录入区收成单个浅色工作台，科目、归属、计划参考、金额和确认动作在一块完成，不再用厚重黑块去抢注意力。
- 支出录入顶部的科目摘要卡统一改成单行展示，桌面端优先保留横向空间给金额、日期和确认动作。
- `退费退款` 已改到收入侧的“其他收支”入口，不再出现在支出挂账科目里。
- 账本记录区保留单行台账样式，摘要继续显示原有的提成等信息，金额单列展示；只允许直接修改数量或金额，不再要求先作废重录；作废会直接执行，已作废分录可在历史里直接取消作废。
- 账本记录补了带日历感的单日 / 单周筛选，可以按某一天或某一周过滤历史分录。
- `记一笔` 和 `账本记录` 的输入框字重恢复为常规值；账本记录表头统一居中，作废行摘要恢复到表格基准字号，不再因为划线而放大。
- 团员收入台账头部的 `业务发生日` 胶囊已和右侧摘要胶囊统一到同一高度；桌面端表头统一回到中线，`团员` 列内容做水平居中，列宽按内容分配，不再把后 7 列强行做成等宽。
- `其他收支` 已去掉“补充对方与备注”按钮；`挂账科目 / 业务发生日 / 金额 / 确认入账 / 对方单位 / 备注` 默认直接显示，桌面端按“标题一行、控件一行”排成同一条横向录入带，金额输入带 `￥` 前缀，前两列也收窄到更紧凑的宽度，空间不足时再自动折行。
- `记支出` 的单笔录入也同步改成桌面端“标题一行、控件一行”的横向录入带，移除了中间说明文案，把 `业务发生日 / 金额 / 对方单位 / 备注 / 确认入账` 收到同一条工作流里，金额输入同样带 `￥` 前缀。
- 已过账历史改成高密度账单表格，并补了方向 / 状态 / 关键词筛选；桌面端保持单行台账，移动端保持紧凑卡片。
- 各工作台卡片的页头统一压成紧凑首行，`eyebrow + 标题`、方向切换、月份筛选、顶部摘要和主操作会优先并进同一行；空间不足时再自动换行，避免首屏被标题区吃掉太多高度。
- `收入引擎` 的图例和“同步默认”也并进标题首行，不再单独占下一排。
- `收入引擎` 和成本编辑里的 `训练与补充` 都改成桌面端两个月并排、每个月内部保持单行；恢复默认收成月份格右上角的圆形按钮，默认模板行继续单独编辑，不再用文字按钮额外占位。
- 顶部摘要卡和控制卡统一收成单行信息条，优先使用“标签在左、数值或控件在右”的结构，不再把标签和值硬拆成上下两层。
- 版本管理弹层收成窄面板，操作、状态和分享链接都在一屏内完成，避免横向和双滚动条。
- 预实分析不再只堆数字卡片，补齐了趋势图、偏差驱动和科目对账。
- 预实分析默认聚焦有差异的科目，并补了收入达成、成本执行、净经营偏差和首要原因四个复盘入口。
- 公开分享页现在包含经营分析、月度结果、成员表现和模型输入四个只读视图。
- 回滚、删除、重置、撤销分享、注销账号等高风险动作都会先确认，再执行；账本分录的作废改为直接执行，历史里支持取消作废。
