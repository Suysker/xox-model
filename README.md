# xox-model

基于 `Vite + React + TypeScript + Tailwind` 的地下偶像团体 ROI / 现金流建模工具。

这版产品不是把所有参数摊平，而是按用户真正会编辑的业务对象来组织：

- 左侧主导航：`经营分析 / 模型输入`
- 右上角工作区：`保存快照 / 发布版本 / 版本库 / 导入 / 导出`
- 输入模块按业务拆成：`股东投资 / 收入引擎 / 成本结构`

## 产品结构

### 经营分析

- 三档场景：`悲观 / 基准 / 乐观`
- 场景走势：查看 `累计现金 / 月利润 / 月收入 / 月总成本`
- 经营明细：按月份看当前月输入、成本结构、利润和回本进度
- 月度表：逐月展开营收、线上营收、成本、利润、累计现金
- 成员贡献拆解：成员折线图 + 当前月环图

### 模型输入

- 股东投资
  - 录入股东名称、投资金额、分红比例
  - 在这里设置经营开始月份、规划月数这类项目级前提假设
  - 回本和 ROI 按股东总投资计算
- 收入引擎
  - 分开录入线下单价、线上单价
  - 配置成员卖张能力、提成、底薪、路费
  - 在同一工作流里配置场次节奏、销售系数、线上系数
  - 默认值直接放在表格第一行，逐月月份在下方联动编辑
  - `线下收入 = 成员单场张数 × 场次 × 销售系数 × 线下单价`
  - `线上收入 = 成员单场张数 × 场次 × 销售系数 × 线上系数 × 线上单价`
  - `总营收 = 线下收入 + 线上收入`
- 成本结构
  - 顶部 `成本概览` 用按月堆叠柱状图看每个月的成本结构，直接展开到化妆、推流、聚餐、耗材这类末级成本项
  - `训练与补充` 负责逐月调整排练、老师和补充支出
  - `专项与耗材` 按列新增 `按月 / 按场 / 按张` 成本，并逐月录入差异

## 默认业务假设

默认数据按对话整理成一个启动期到稳定期的模板：

- 股东投资：A `30000` / B `30000` / C `25000`
- 分红比例：`40% / 35% / 25%`
- 线下单价：`88`
- 线上单价：`120`
- 1 名底薪成员：`1500` 底薪 + `15%` 提成
- 其余成员：`35%` 提成
- 2 名员工：`300 / 场`
- 成员路费按成员逐个配置，财务口径计入 `每场成本`
- 经营起始月份：`3 月`
- 规划周期：默认 `6 个月`
- 3 月：启动月，销售系数偏低，训练频次更高
- 4 月：按对话中的稳定月口径建模
- 5 月起：可开始计入耗材，并叠加增长期节奏
- 线上系数：支持按月调线上收入相对线下月张数的放大比例

这些只是默认模板，真实回本周期要按你的成员结构、活动排期和专项支出重录。

## 使用方式

1. 在 `模型输入 -> 股东投资` 里录入股东金额、分红比例，以及经营开始月份、规划月数。
2. 在 `模型输入 -> 收入引擎` 里设置线上/线下单价、成员卖张能力、场次曲线、销售系数曲线和线上系数。
3. 在 `模型输入 -> 成本结构 -> 成本概览` 里先看按月堆叠柱状图；鼠标悬停看当月 tooltip 明细，点击月份确认当前查看月。
4. 在 `模型输入 -> 成本结构 -> 成本编辑` 里切到 `训练与补充 / 专项与耗材`，调整少数例外月份。
5. 只在 `训练与补充` 和 `专项与耗材` 里改少数特殊月份，例如上新歌、VJ、推流、聚餐，以及耗材的月度计入系数。
6. 回到 `经营分析` 查看三档场景、走势、月度表和成员贡献拆解。
7. 阶段版本形成后，用右上角工作区做快照或发布版本。

## 当前边界

- 仍是单团模型，还没有多团并行
- 分红比例目前用于资金结构录入，还没有展开成“回本后按股东分红”的现金流模拟
- 成员成长仍以单场张数和月销售系数建模，尚未做更复杂的长期成长曲线
- 还没有税务、渠道分账、场地方抽成等更财务化口径

## 本轮更新

- 把原先分散的股东、收入和月度节奏录入重新收敛成 `股东投资 / 收入引擎 / 成本结构`
- 把月份规划前提上移到 `股东投资`，收入页只负责真正的收入驱动项
- 收入底盘拆成 `线下单价 + 线上单价`

- 收入节奏和成本差异都改成“默认值第一行 + 下方逐月行”，不再单独占卡片
- `成本结构` 先看顶部 `成本概览` 的按月堆叠柱状图，再在 `成本编辑` 里切 `训练与补充 / 专项与耗材`
- 月度表新增 `线上营收` 列，经营明细也会拆出 `线下营收` 与 `线上营收`
- 总览页把主走势与单月对比重新收成一张黑底分析区，右侧经营明细改为紧凑指标行，避免左右两栏节奏失衡
- 成员拆解页改成“上方图表分析 + 下方紧凑明细表”，不再用长卡片堆出整页空白
- 成员拆解页的月份滑条下移到 `成员明细` 区，只联动 `成员营收占比` 与明细表；趋势图保持看整段周期
- 成员明细表把 `类型` 从成员列里拆出来，便于直接横向比较底薪/兼职差异
- 成员营收趋势图本身也能点月份；点图上的月份后，右侧占比和下方明细会一起切到该月份
- 成员明细区的标题直接显示为 `XX月 成员明细`，不再重复放右上角月份角标
- 成员营收趋势图的 Y 轴按当前数据范围自适应，不再强制从 `0` 起，避免点位和标签挤在上沿
- 总览页右侧 `经营明细` 已移除 `当前月成本结构` 二次展示，成本信息只保留在 `成本拆解` 内，避免同一层级重复
- 总览页左侧 `场景走势` 已切回浅底图表风格，和其它分析卡统一，不再用整块黑底做主图
- 总览页 `经营明细` 标题已经包含月份，因此右上角不再重复显示当前月份，只保留周期索引
- 总览页左右分栏重新分配宽度，`经营明细` 侧栏加宽；明细行标签也改成更适合中文的单行样式，尽量避免成本项和输入项出现难看的换行
- 总览页继续把左侧主图区放宽，并缩小两栏与右栏内部的间隙，优先把宽度留给图表和经营明细内容本身
- 总览页的图表指标切换不再用大胶囊导航样式，而是改成更轻的 `图表口径` 切换器，并把文案收短成 `累计现金 / 利润 / 收入 / 总成本`
- `成本结构` 已从三个写死标量改成真正的成本项清单：按 `每月固定 / 每场固定 / 每张成本` 三类分别新增、改名、删项；这里不再出现“默认值”输入框
- 旧工作区里的 `monthlyFixedCost / perEventOperatingCost / materialCostPerUnit` 会自动迁移到新的成本数据结构，不会因为模型升级丢掉历史数据
- `成本编辑` 现在只保留 `训练与补充 / 专项与耗材` 两类月度差异
- `成本概览` 已收成单一的按月堆叠柱状图，每个月独立成柱，按末级成本项分色堆叠并显示总额标签；hover 会浮出当月成本明细 tooltip
- `成本概览` 的月总额标签会紧贴柱顶显示；hover tooltip 会贴近当前柱子，并补充每个成本项占当月总成本的占比
- `成本编辑` 里的 `同步默认` 已下沉到各自表头：训练表单独同步，专项表则放在 `添加成本列` 左侧，不再占全局头部
- `成本编辑` 已去掉额外的 `月度差异` 层级和说明文案，两个 tab 直接放在标题右侧，下面直接显示当前表格
- `成本编辑` 现在默认打开 `专项与耗材`；tab 贴着标题放在左侧，右侧则只保留当前 tab 的动作按钮。训练表里的 `额外每场 / 额外固定` 已删除，统一改由 `专项与耗材` 维护
- 2026-03-09 `专项与耗材表` 已统一成列驱动模型：每一列都是一个可编辑成本项，列头可直接新增、改名，并切换 `按月 / 按场 / 按张`
- `按场` 和 `按张` 都收成单单元格里的复合输入：前者是 `单价 / 场次`，后者是 `单价 / 系数`；`按张` 的月份仍按 `0-1` 系数录入，例如 `3月耗材 = 0`、`5月耗材 = 1`
- `按场` 复合格里的 `场次` 输入会刻意更窄；逐月恢复动作也改成纯图标按钮，减少密集表格里的横向浪费
- `专项与耗材表` 会按费用类型自动排序成 `按张 -> 按场 -> 按月`，先把最常改、最需要对比的列放在左侧
- 成本表里的数字输入会隐藏浏览器原生上下按钮，避免窄输入框里数字被 spinner 挤掉
- 专项成本表里的金额输入不再限制固定步进，像 `150`、`300` 这类合法费用可以直接录入，不会再被浏览器标成无效。
- `耗材` 不再是独立硬编码列，而是普通成本列里的 `按张` 类型；名称本身不再带 `/月 /场 /张` 后缀，计费方式都放在列头下拉里
- 历史工作区如果还带旧的独立 `耗材/张` 字段或带后缀的专项名称，导入时会自动合并成新的 `按张` 成本列，并去掉名称后缀
- 导入旧工作区时，历史的 `vjCost / makeupPerEventCost / streamingPerEventCost / mealPerEventCost` 会自动迁移成新的专项成本项和月度计入次数，不需要手动重录。
