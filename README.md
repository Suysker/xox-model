# xox-model

一个基于 Vite + React + TypeScript + Tailwind 的地下偶像团 ROI 建模工作台。

这版的重点不是“把参数都露出来”，而是把用户操作路径做顺：

- 左侧导航：`经营分析` / `模型输入`
- 右上角工作区动作：保存快照、发布版本、版本库、导入、导出
- 月度排期改成多月矩阵，而不是逐月点开
- 周期本身可配：经营开始月份、规划月数

## 产品结构

### 经营分析

- 三档场景卡：悲观 / 基准 / 乐观
- 上下界走势图：月收入 / 月总成本 / 月利润 / 累计现金
- 月度经营结果表：逐月看收入、成本、利润、累计现金、回本状态
- 成员贡献拆解：按月份看每个成员的收入贡献

### 模型输入

- 成员与员工：成员单独配置提成、底薪和三档单场张数；员工单独配置岗位、月薪和场次成本
- 经营底盘：前期投入、单张售价、基础固定成本、基础场次成本、耗材成本，以及经营开始月份、规划月数
- 月度排期：先调一行默认模板，再一键应用到全部月份，最后逐月维护差异

### 工作区

工作区不再单独占一个主 tab，而是收在页面右上角：

- 保存快照
- 发布版本
- 版本库展开 / 收起
- 导出 JSON
- 导入 JSON

## 默认业务假设

默认数据按你给的对话整理成一个“启动 -> 稳定 -> 增长”的样板：

- 前期已投入：85,000 元
- 单张售价：88 元
- 1 名底薪成员：1,500 元底薪 + 15% 提成
- 2 名场务员工：300 元 / 场
- 其余成员：35% 提成
- 经营从 3 月开始，默认规划 6 个月
- 3 月：启动月，销售系数 0.66，排练 8 次，老师 8 次，不计耗材
- 4 月：稳定月，销售系数 1.0，排练 4 次，老师 4 次，不计耗材
- 5 月起：增长月，销售系数 1.3 到 1.35，耗材开始计入

这些只是样板。真实回本节奏要按你的成员结构、场次排期和专项支出重新录入。

## 本地运行

Windows PowerShell 下建议用 `npm.cmd`：

```bash
npm.cmd install
npm.cmd run dev
```

## 验证

```bash
npm.cmd run test
npm.cmd run build
```

预期结果：

- `npm.cmd run test` 通过，覆盖月度模型和工作区导入导出 / 旧配置迁移
- `npm.cmd run build` 通过，说明类型和前端构建正常

## 使用方式

1. 在左侧切到 `模型输入 -> 成员与员工`，先录成员结构、三档单场张数，以及员工月薪 / 场次成本
2. 在 `模型输入 -> 经营底盘`，录前期投入、售价和基础成本
3. 在 `模型输入 -> 经营底盘`，设置经营开始月份和规划月数
4. 在 `模型输入 -> 月度排期`，先调整默认模板，再一键应用到全部月份
5. 只对少数特殊月份做逐月微调和备注
6. 回到 `经营分析`，看三档场景、走势图、月度表和成员拆解
7. 阶段性版本形成后，用右上角保存快照或发布版本

## 当前边界

- 仍然是单团模型，没有多团并行
- 成员成长仍按三档单场张数建模，还没有逐月成长曲线
- 还没有税务、渠道分账、场地方抽成等更财务化项目
- 手机端月度矩阵仍以横向滚动为主，后续最值得补的是专门的移动端卡片视图

## 本轮更新

- 版本管理不再占用主内容顶部整行，改成右上角悬浮历史按钮；点击后以右侧抽屉打开工作区、导入导出和版本列表。
- 成本口径已校正：`总成本` 和 `月总成本` 现在都包含提成，和成本拆解、利润、图表口径一致。
- 顶部概览把无关的版本信息替换成 `平均单场`，更贴近经营判断。
- 团队成员表改成固定列宽，单位移到列表头，不再在每个单元格里重复显示 `% / 元 / 张`，桌面视图更紧凑。
- 经营开始月份和规划月数已移动到经营底盘，不再和月度排期混在一起。
- 月度排期改成“默认模板 + 逐月差异”的工作流：先调一行默认值，一键批量应用，再对个别月份做覆盖。
- 总览页的 `场景走势 / 经营明细` 改成顶对齐布局，走势图不再被右侧明细卡片拉出大片空白。
- `经营明细` 的月份切换改成单行滚动轨道，并在标题区单独标出当前查看月份，避免月份按钮挤成两行。
- `300 元 / 场` 这类执行成本不再挂在成员身上，模型里新增了独立的员工配置表，成员收入和员工成本分开建模。
- 总览页不再依赖横向滚动：走势图改成自适应宽度并按月份数量自动稀释标签，经营明细里的月份切换改成无滚动的月历式按钮矩阵。
