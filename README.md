# xox-model

一个基于 Vite + React + TypeScript + Tailwind 的地下偶像团 ROI 建模工作台。

这版不是“堆输入框”的 demo，而是围绕地下偶像团的真实经营逻辑，拆成三个稳定层：

- 团队成员：成员数量、成员类型、提成、底薪、场补、悲观 / 基准 / 乐观单场张数
- 经营底盘：前期投入、单张售价、基础固定成本、基础场次成本、耗材成本
- 月度排期：每个月的场次、销售系数、排练、老师、额外场次成本、额外固定成本、专项成本、备注

## 当前产品形态

页面分成 3 个主标签：

- `经营分析`
  - 三档场景卡片：悲观 / 基准 / 乐观
  - 上下界走势图：月收入 / 月总成本 / 月利润 / 累计现金
  - 月度经营结果表：逐月看收入、成本、利润、累计现金、回本状态
  - 成员贡献拆解：按月份看每个成员的单场张数、月总张数、提成和留存
- `模型输入`
  - 团队成员 tab
  - 经营底盘 tab
  - 月度排期 tab
- `版本工作区`
  - 保存快照
  - 发布版本
  - 导出 JSON
  - 导入 JSON
  - 本地版本管理

## 默认业务假设

默认数据按你给的对话整理成一个“启动 -> 稳定 -> 增长”的样板：

- 前期已投入：85,000 元
- 单张售价：88 元
- 1 名底薪成员：1,500 元底薪 + 15% 提成
- 2 名兼职成员：300 元 / 场补
- 其余成员：35% 提成
- 3 月：启动月，销售系数 0.66，排练 8 次，老师 8 次，不计耗材
- 4 月：稳定月，销售系数 1.0，排练 4 次，老师 4 次，不计耗材
- 5 月起：增长月，销售系数 1.3 到 1.35，耗材开始计入

这些只是样板，不代表真实结论。真实回本周期要按你的成员、场次和专项支出重新录。

## 本地运行

Windows PowerShell 下建议用 `npm.cmd`：

```bash
npm.cmd install
npm.cmd run dev
```

## 验证

```bash
npm.cmd run test
npm.cmd run build
```

预期结果：

- `npm.cmd run test` 通过，覆盖月度模型和工作区导入导出解析
- `npm.cmd run build` 通过，说明类型和前端构建正常

## 使用方式

1. 先在 `模型输入 -> 团队成员` 里录入成员结构和三档单场张数
2. 在 `模型输入 -> 经营底盘` 里录入前期投入、售价和基础成本
3. 在 `模型输入 -> 月度排期` 里逐月配置场次、销售系数和专项成本
4. 回到 `经营分析` 看三档总览、走势图、月度表和成员拆解
5. 形成一个阶段性口径后，在 `版本工作区` 保存快照或发布版本
6. 需要分享或备份时，直接导出 JSON；需要恢复时，导入 JSON

## 本次重构改了什么

- 把超长 `App.tsx` 拆成多个组件，壳层只负责状态和流程编排
- 把主导航和输入层改成 tab 化，减少无效纵向堆叠
- 把团队假设改成多维表格，而不是零散卡片
- 增加了月度经营结果表和成员贡献拆解
- 增加了工作区版本能力：快照、发布版、导入、导出
- 增加了工作区解析测试，保证导入文件格式有校验

## 当前边界

- 仍然是单团模型，没有多团并行
- 成员成长仍按三档单场张数建模，还没有逐月成长曲线
- 还没有税务、分账渠道、场地方抽成等更财务化项

如果继续往下做，最值得加的是：

- 多团并行
- 成员成长曲线
- 收入类型拆分
- 手机端专门的卡片式月度结果视图
